# 可验证抽奖 — 社交平台抽奖工具

> **这不是区块链项目，不是 Web3 工具，不是 NFT mint，不是企业级随机数服务。** 这是一个给 X/Twitter 博主用的社交平台抽奖工具，仅此而已。

**一句话：博主设截止时间 → 粉丝参与 → 截止后用 drand 随机数开奖 → 结果公开可验证。**

### 它能做什么（范围很小）

- ✅ X/Twitter 博主举办评论/点赞抽奖
- ✅ 任何社交平台上的公开名单抽奖

### 它不能做什么（很重要）

- ❌ 不是智能合约，不跑在链上
- ❌ 不是 NFT mint / 白名单抽取工具
- ❌ 不是企业级随机数生成服务
- ❌ 不存储任何用户数据

---

① 填参数 → ② 发链接 → ③ 截止时间到 → ④ 打开同一链接点"开奖" → ⑤ 把结果发出去

## 三个阶段的完整流程

---

### 阶段一：宣布抽奖（博主操作）

博主在社交平台发布抽奖活动，同时附上抽奖链接。

**博主需要做：**

1. 打开 [drand-draw.pages.dev](https://drand-draw.pages.dev) → 点击「发起抽奖」
2. 选择 drand 网络（推荐 **QuickNet**，3 秒一轮，出块时间精确可算）
3. 填写 **参与人数 N**（你预期有多少人参加）
4. 填写 **截止时间**（精确到分钟，选择时区）
5. 设置 **奖项**（默认一等奖 1 人，可添加更多奖项）
6. 点击「生成抽奖链接」

你会得到两样东西：

| 类型 | 示例 | 用途 |
|------|------|------|
| 🔗 **链接** | `drand-draw.pages.dev/#/?chain=quicknet&deadline=1715000000&n=100&prizes=1` | 直接发在社交平台，粉丝点开即用 |
| 🔢 **短码** | `q-66364280-2s-1` | 怕外链被限流？发短码，让粉丝到网站输入 |

**将链接/短码公开发布在你的抽奖推文中。**

> ⚠️ 重要：必须在截止时间**前**发布链接。截止后再发，粉丝无法确认 deadline 是否被篡改。

**💡 实用技巧：用 X 评论 + Grok 自动获取参与列表**

在 X 上举办抽奖时，可以要求用户**评论**（而非点赞）来参与。理由：

1. X 的评论区是公开可查的，任何人都可以核对名单
2. **Grok**（X 内置 AI）可以读取帖子的评论数据
3. 博主只需对 Grok 说：*"列出这条帖子的所有评论用户"* → 瞬间拿到完整列表
4. 无需手动截图、无需爬虫、全程可公开验证

配套话术建议：
```
🎉 抽奖规则：
• 截止前评论本推文即可参与
• 截止后我会用 Grok 获取评论列表
• 按字母排序后编号
• 用 drand 随机数开奖
• 任何人都可独立验证

验证链接: drand-draw.pages.dev
```

这样既降低了博主的工作量，又增加了透明度——因为评论列表是公开的，粉丝自己也能核对。

**此时粉丝打开链接会看到：**
```
┌──────────────────────────────────┐
│  🔒 暂未开奖                      │
│                                    │
│  截止时间: 2026-05-10 18:00 UTC+8 │
│  ⏳ 距离截止还有 2天 3小时 15分    │
│                                    │
│  参与人数: N = 100                  │
│  奖项: 一等奖 × 1                   │
│  预计使用 drand Round #1234567     │
└──────────────────────────────────┘
```

**可验证的信息：**
- 截止时间是否和博主说的一致
- N 是否和博主说的一致
- 奖项设置是否和博主说的一致

---

### 阶段二：公布候选列表（博主操作）

截止时间到了，博主需要：

1. **收集所有有效参与者**（用 Grok 获取评论用户列表）
2. **去重**：同一用户多次互动只算一次
3. **排序**：将所有 @username 按 ASCII 码升序排列（数字 → 大写字母 → 小写字母）
4. **编号**：从 0 开始依次编号
5. **公开列表**：将完整编号列表（或截图）公开发布

> 💡 **实用工具**：在「发起抽奖」页面下方有「候选列表排序工具」。把 handle 粘贴进去（支持空格/逗号/换行分隔），自动排序编号，可以一键复制排序文本，或导出为图片直接发推。支持「一行一列」和「尽量正方形」两种图片模式。

> ⚠️ 如果有参与者发现自己不在列表上，此时应提出异议。

## 为什么这是不可伪造的？

### 攻击面分析

| 博主想做什么？ | 能成功吗？ | 为什么？ |
|---|---|---|
| 先看到随机数再选 round | ❌ 不行 | Round 由截止时间唯一确定，deadline 在链接中锁定 |
| 对 A 说 N=100，对 B 说 N=200 | ❌ 不行 | N 在链接中锁定，所有人都看到同一个数 |
| 用不同的排序规则 | ❌ 不行 | 排序规则由页面硬编码（ASCII 升序），博主无法篡改 |
| 对同一抽奖多次开奖直到满意 | ❌ 不行 | 同一个 deadline 只能算出一个结果 |
| 截止后篡改候选名单 | ⚠️ 需要社会约束 | 名单在阶段二公开过，粉丝可以核对 |

### 技术上为什么可信

drand 是一个**去中心化随机信标网络**，由以下机构共同运行：

- Cloudflare
- Protocol Labs
- 以太坊基金会
- ChainSafe
- 以及其他 10+ 独立机构

没有任何一个实体可以操控随机数。drand 的输出是公开的、可验证的、不可预测的。

### 还需要注意的（社会约束）

1. **博主必须在截止前公开链接** — 否则无法证明 deadline 没有被改过
2. **博主必须在开奖前公开完整候选列表** — 粉丝需核对名单是否正确
3. **列表去重由博主负责** — 同一用户多次互动计为一次，这需要博主诚信

## 常见问题

### Q: 短码和链接有什么区别？

没区别，包含的信息完全一样。链接更方便（点开即用），短码不怕 X 限流外链。

### Q: 为什么推荐用 QuickNet 而不是 Default？

QuickNet 每 3 秒出一个随机数，Default 每 30 秒出一个。截止后 QuickNet 最多等 3 秒就能开奖，Default 最多等 30 秒。

### Q: 如果开奖时 drand 网络连不上怎么办？

页面会自动切换到备用节点（api2.drand.sh、drand.cloudflare.com 等），如果全部失败会提示重试。

### Q: 我可以在截止前知道中奖结果吗？

不能。drand 的随机数在对应 round 出块前是不可知的。而 round 在截止时间之后才会产生。

### Q: 如果两个奖位算出相同编号怎么办？

系统会自动顺延到下一个未占用的编号。所有奖位独立计算，互不影响。

### Q: 这个工具会存储我的数据吗？

不会。所有信息都在链接或短码中，页面是纯前端的，没有任何后端服务器。

## 算法规范（Algorithm Specification）

> 以下规范供开发者、安全审计者独立复现抽奖结果。你可以用任何语言（Python、JavaScript、Go 等）实现本算法，得到的结果应与本页面一致。

---

### 1. 参与列表排序与编号

```
输入:  参与者的社交平台用户名列表（字符串数组）
输出:  编号 0 到 N-1 的有序列表

步骤:
  1. 去重：移除重复的用户名（同一用户多次互动只计一次）
  2. 排序：按字符串的 ASCII 码升序排列
     - 规则：标准字典序，逐字节比较 UTF-8 编码
     - 示例排序: "0" < "9" < "@" < "A" < "Z" < "_" < "a" < "z"
  3. 编号：从 0 开始，依次分配
     - 第 1 名用户 → 编号 0
     - 第 2 名用户 → 编号 1
     - 第 N 名用户 → 编号 N-1

N = 有效参与用户总数
```

---

### 2. Round 计算

```
输入:  deadline（Unix 时间戳，秒）, chain配置
输出:  round 编号（正整数）

对每条 drand 链，有固定的创世时间和出块间隔:

┌──────────┬──────────────────────────────────────────┬────────────┬──────────────┐
│ 链       │ chain_hash                              │ genesis_ts │ period(秒)   │
├──────────┼──────────────────────────────────────────┼────────────┼──────────────┤
│ quicknet │ 52db9ba70e0cc0f6eaf7803dd07447a1f5477735 │ 1692803367 │ 3            │
│          │ fd3f661792ba94600c84e971                 │            │              │
├──────────┼──────────────────────────────────────────┼────────────┼──────────────┤
│ default  │ 8990e7a9aaed2ffed73dbd7092123d6f2899305 │ 1595431050 │ 30           │
│          │ 40d7651336225dc172e51b2ce                │            │              │
├──────────┼──────────────────────────────────────────┼────────────┼──────────────┤
│ evmnet   │ 04f1e9062b8a81f848fded9c12306733282b2727 │ 1727521075 │ 3            │
│          │ ecced50032187751166ec8c3                 │            │              │
└──────────┴──────────────────────────────────────────┴────────────┴──────────────┘

算法:

  elapsed = deadline - genesis_ts
  round   = floor(elapsed / period) + 1

  即：截止时间之后的第 1 个未出块的 round。

示例 (quicknet, deadline = 1715000000):
  elapsed = 1715000000 - 1692803367 = 22196633
  round   = floor(22196633 / 3) + 1 = 7398878
```

---

### 3. 获取随机数

```
输入:  chain_hash, round
输出:  randomness（64字符十六进制字符串）

通过 drand HTTP API 获取:

  GET https://api.drand.sh/{chain_hash}/public/{round}

响应格式:
  {
    "round": 7398878,
    "randomness": "a3f25c8d1e7b...",  // ← 64字符十六进制
    "signature": "92daf574..."
  }

注意:
  - v1 API 直接返回 randomness 字段
  - v2 API 只返回 signature，需自行计算 randomness = SHA-256(signature)
  - 本页面使用 v1 API (/{chain_hash}/public/{round}) 以获得兼容性
  - 建议从多个 relay 获取并比对结果（api.drand.sh, api2.drand.sh, drand.cloudflare.com）
```

---

### 4. 中奖计算

```
输入:  randomness（64字符十六进制）, N（参与人数）, prizeTiers（各奖项人数数组）
输出:  winners（中奖编号数组，长度为 sum(prizeTiers)）

算法:

  function computeWinners(randomness, N, prizeTiers):
    winners = []
    used = {}        // 已占用的编号，用于碰撞处理

    for i = 0 to len(prizeTiers) - 1:          // 遍历奖项层级
      for j = 0 to prizeTiers[i] - 1:          // 遍历该层级的每个奖位
        shift = len(winners)                   // 当前已产生的奖位数
        seedHex = SHA256(randomness + ':' + shift)  // 哈希派生种子
        bigVal = parseBigInt(seedHex, 16)      // 转为大整数
        idx = bigVal % N                        // 取模得到候选编号

        while idx in used:                     // 碰撞处理
          idx = (idx + 1) % N                  // 顺延到下一位

        used.add(idx)
        winners.append(idx)

    return winners

说明:
  - 每个奖位使用 SHA-256 哈希派生种子：SHA256(randomness + ':' + shift)
  - shift = 0, 1, 2, ... 分别对应第 1、2、3... 个奖位
  - 哈希确保了不同 shift 产生的种子互不相关，不会出现连续编号
  - 碰撞时顺延到下一个未被占用的编号（环形查找）
  - 所有奖位计算完成后，winners 数组的顺序即为各奖项中奖编号

示例 (randomness = "a3f25c...", N = 100, prizeTiers = [1, 3]):

  一等奖:
    shift = 0, seedHex = SHA256("a3f25c...:0")
             = "e7b2a9..._a_random_256bit_hash"
    bigVal = 0xe7b2a9...
    idx = bigVal % 100 = 42                → #42 中一等奖

  二等奖 (第 1 个):
    shift = 1, seedHex = SHA256("a3f25c...:1")
             = "8d3f1c..._completely_different_hash"
    bigVal = 0x8d3f1c...
    idx = bigVal % 100 = 15               → #15 中二等奖

  二等奖 (第 2 个):
    shift = 2, seedHex = SHA256("a3f25c...:2")
    bigVal = ...
    idx = bigVal % 100 = 78               → #78 中二等奖

  二等奖 (第 3 个):
    shift = 3, seedHex = SHA256("a3f25c...:3")
    bigVal = ...
    idx = bigVal % 100 = 42               → 碰撞! 顺延至 43

    used = {42, 15, 78}, 43 可用         → #43 中二等奖

最终结果:
  一等奖: #42
  二等奖: #15, #78, #43
```

### 5. 验证步骤（独立复现）

要独立验证一个抽奖结果，你不需要任何专用工具，只需以下 5 步：

```
1. 确认 deadline 和 N
   检查博主公布的 deadline 和 N 是否与抽奖链接中的一致。

2. 计算 round
   round = floor((deadline - genesis_ts) / period) + 1

3. 获取 randomness
   用 curl 或任意 HTTP 客户端:
   curl https://api.drand.sh/{chain_hash}/public/{round}
   取出响应中的 "randomness" 字段。

4. 计算中奖编号
   用你熟悉的语言实现上面的 computeWinners 函数。
   传入 randomness、N、prizeTiers。

5. 比对
   你算出的 winners 是否与博主声明的一致？
   是 → 验证通过 ✅
   否 → 验证不通过 ❌
```

#### 命令行验证示例

```bash
# 以 quicknet 链为例
CHAIN_HASH="52db9ba70e0cc0f6eaf7803dd07447a1f5477735fd3f661792ba94600c84e971"
ROUND=7398878
N=100

# 获取随机数
RANDOMNESS=$(curl -s "https://api.drand.sh/${CHAIN_HASH}/public/${ROUND}" | python3 -c "import sys,json; print(json.load(sys.stdin)['randomness'])")

echo "Round: $ROUND"
echo "Randomness: $RANDOMNESS"
echo "N: $N"

# Python 计算中奖编号
python3 -c "
randomness = '$RANDOMNESS'
N = $N
prizes = [1, 3]  # 一等奖1人，二等奖3人

# 新版（使用 SHA-256，每个奖位独立随机）
import hashlib
winners = []
used = set()
for i, count in enumerate(prizes):
    for j in range(count):
        shift = len(winners)
        seed = int(hashlib.sha256((randomness + ':' + str(shift)).encode()).hexdigest(), 16)
        idx = seed % N
        while idx in used:
            idx = (idx + 1) % N
        used.add(idx)
        winners.append(idx)

print(f'Winners: {winners}')
"
```

### 6. 短码编码规范

短码格式为用 `-` 分隔的字段：

```
{chain字首}-{deadline十六进制}-{N的base36}-{prizes的base36}-{winners的base36}

字段说明:
  chain字首:   q = quicknet, d = default, e = evmnet
  deadline:    Unix 时间戳的小端十六进制（不含 0x）
  N:           base36 编码（0-9a-z）
  prizes:      逗号分隔的 base36 数字（可选）
  winners:     逗号分隔的 base36 数字（可选）

示例:
  q-66364280-2s-1,3           ← 仅配置，未开奖
  q-66364280-2s-1,3-2a,f,4e  ← 已开奖
```

### 8. 安全边界

| 参数 | 最大值 | 说明 |
|------|--------|------|
| N | 1,048,575 | 20 位二进制，约 100 万参与者 |

| 短码长度 | ~40 字符 | 取决于奖项和中奖人数 |
