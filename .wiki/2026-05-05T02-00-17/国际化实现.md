# 国际化实现

整个应用的 UI 文本支持中英双语，切换时无需重新加载页面，即时生效。这套系统的核心代码全部集中在 `src/i18n.js` 中——一个无依赖、约 100 行的独立模块。

## 架构概览

```
┌─────────────────────────────────────────────┐
│                  i18n.js                      │
│                                               │
│  LANGUAGES = ['zh', 'en']                     │
│                                               │
│  detectLang()  ─── localStorage ──→ navigator │
│  setLang(lang) ─── localStorage + <html lang> │
│  getLang()     ─── currentLang               │
│                                               │
│  T = { zh: { ... }, en: { ... } }             │
│       ↑ 翻译表       ↑ 翻译表                 │
│                                               │
│  t(key, ...args)     tLang(lang, key, ...args)│
└─────────────────────────────────────────────┘
       │              │              │
       ▼              ▼              ▼
   CreateDraw.js  DrawStatus.js  Guide.js / SortTool.js
      (t)            (t)          (getLang)
```

所有组件只通过 `t()` 或 `getLang()` 两个函数与 i18n 系统交互，不直接访问翻译表。 [来源](src/i18n.js#L1-L162)

## 语言检测：`detectLang()`

检测逻辑分两步，优先级从高到低：

1. **localStorage** — 检查 `drand-lang` 键，如果有且值在 `LANGUAGES` 白名单内则直接返回
2. **浏览器语言** — 降级到 `navigator.language`，前缀为 `zh` 则返回 `'zh'`，否则返回 `'en'`

```js
export function detectLang() {
  const stored = localStorage.getItem('drand-lang')
  if (stored && LANGUAGES.includes(stored)) return stored
  const nav = (navigator.language || '').toLowerCase()
  return nav.startsWith('zh') ? 'zh' : 'en'
}
```

这意味着：用户一旦手动切换过语言，后续访问都会记住偏好；首次访问的访客则根据浏览器语言自动适配。 [来源](src/i18n.js#L5-L10)

## 语言切换：`setLang()` 与切换按钮

`setLang(lang)` 做三件事：

```js
export function setLang(lang) {
  currentLang = lang
  localStorage.setItem('drand-lang', lang)
  document.documentElement.lang = lang === 'zh' ? 'zh-CN' : 'en'
}
```
- 更新内存中的 `currentLang`
- 将偏好持久化到 `localStorage`
- 设置 `<html lang="...">` 属性，辅助无障碍技术

切换入口在 `main.js` 顶栏的**地球图标按钮**。点击时读取当前语言，取反后调用 `setLang()` 然后重新执行 `render()`，整个页面即刻以新语言重绘：

```js
app.querySelector('#lang-switch').addEventListener('click', () => {
  const newLang = getLang() === 'zh' ? 'en' : 'zh'
  setLang(newLang)
  render()
})
```

注意是**全量重渲染**而非局部更新——这确保了所有文本一致切换，无须逐元素追踪状态。 [来源](src/main.js#L74-L78) [来源](src/i18n.js#L12-L16)

## 翻译表：`T` 对象

`T` 包含两个子对象——`T.zh` 和 `T.en`，每个都是一个扁平的键值映射。两表**键名完全一致**，确保调用方无须感知当前语言：

```js
export const T = {
  zh: {
    siteTitle: '可验证抽奖',
    siteDesc: '基于 drand 去中心化随机数网络的公开可验证抽奖工具',
    tabCreate: '发起抽奖',
    // ...
  },
  en: {
    siteTitle: 'Verifiable Lottery',
    siteDesc: 'A public verifiable lottery tool powered by drand decentralized randomness',
    tabCreate: 'Create Draw',
    // ...
  },
}
```

翻译表覆盖了应用中全部面向用户的文本：从标签栏标题、表单标签、按钮文字，到工作原理说明和页脚信息，共约 70 个键。 [来源](src/i18n.js#L20-L154)

## 动态参数翻译：`t()` 和函数类型值

### 基本用法

`t(key)` 根据当前语言取出对应值：

```js
t('siteTitle')        // → '可验证抽奖' 或 'Verifiable Lottery'
t('tabCreate')        // → '发起抽奖' 或 'Create Draw'
```

### 带参数的翻译（函数值）

翻译表中的值不限于字符串，也支持**函数**——这是这套系统最巧妙的设计。以奖项等级为例，中文需要将索引 `0` 映射为"一等奖"、`1` 为"二等奖"，英文则直接显示 `Tier 1`、`Tier 2`：

```js
// zh 翻译表
prizeTier: (i) => `第${'一二三四五六七八九十'[i] || (i+1)}等奖`

// en 翻译表
prizeTier: (i) => `Tier ${i+1}`
```

`t()` 函数在检测到值为函数类型时自动调用它，并将额外参数透传：

```js
export function t(key, ...args) {
  const val = T[currentLang]?.[key] || T.zh[key]
  return typeof val === 'function' ? val(...args) : val
}
```

所以在组件中，调用方无需关心翻译内容是否是动态的：

```js
t('prizeTier', 0)   // → '一等奖' 或 'Tier 1'
t('prizeTier', 2)   // → '三等奖' 或 'Tier 3'
t('prizeTier', 9)   // → '第十等奖' 或 'Tier 10'
```

中文的汉字序号映射只处理到 `0-9`（一至十），超出范围时降级为 `${i+1}等奖`。 [来源](src/i18n.js#L40-L41) [来源](src/i18n.js#L156-L160)

## 语言特定渲染：`getLang()`

对于某些场景，仅靠 `t()` 的键值映射不够灵活——例如需要根据语言决定加载哪份 Markdown 指南文件。此时组件直接调用 `getLang()` 获取当前语言代码：

```js
// Guide.js
const lang = getLang()
const mdFile = lang === 'en' ? '/GUIDE_EN.md' : '/GUIDE.md'
```

SortTool.js 中也有类似用法，一些段落级别的 UI 提示文本因存在条件和结构的差异，直接在组件内通过 `getLang()` 条件渲染，而非塞入翻译表。 [来源](src/components/Guide.js#L3-L5) [来源](src/components/SortTool.js#L9-L11)

## 如何添加新语言

以添加日语 (`ja`) 为例，需要改动 `src/i18n.js` 一处文件，共三步：

### 1. 注册语言代码

在 `LANGUAGES` 数组中追加 `'ja'`：

```js
export const LANGUAGES = ['zh', 'en', 'ja']
```

`LANGUAGES` 被 `detectLang()` 用于校验 localStorage 中的值是否为合法语言，因此新增语言后必须同步更新此数组。 [来源](src/i18n.js#L1)

### 2. 添加完整翻译表

在 `T` 对象中增加 `T.ja` 子对象，键名与 `T.zh` 和 `T.en` **完全一致**：

```js
export const T = {
  zh: { /* ... */ },
  en: { /* ... */ },
  ja: {
    siteTitle: '検証可能な抽選',
    siteDesc: 'drand 分散型乱数ネットワークを利用した公開検証可能な抽選ツール',
    tabCreate: '抽選を作成',
    prizeTier: (i) => `${i+1}等賞`,
    // ... 其余所有键
  },
}
```

**特别提醒**：翻译表中的每个键都必须提供，包括函数类型的键（如 `prizeTier` 和 `prizeTierEn`）。如果漏了某个键，`t()` 的 fallback 机制会自动兜底到中文翻译 `T.zh[key]`，不会崩溃，但会导致界面中英文混杂。 [来源](src/i18n.js#L156-L160)

### 3. 更新语言切换按钮文案

语言切换按钮的文案本身也通过翻译表提供——`langSwitch` 键。中文下显示 `English`，英文下显示 `中文`。新增语言时，需要在**所有现有语言**的翻译表中更新此键，保证切换按钮能正确显示目标语言的名称：

```js
// zh:  langSwitch: 'English',       // 中文界面下按钮显示"English"
// en:  langSwitch: '中文',           // 英文界面下按钮显示"中文"
// ja:  langSwitch: 'English / 中文',  // 日语界面下按钮显示什么？
```

此外，`main.js` 中的语言切换逻辑是二值取反（`zh ↔ en`），新增语言后需要改写为循环切换。 [来源](src/main.js#L74-L78)

## 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 翻译表格式 | 纯 JS 对象，非 JSON | 支持函数类型值，实现动态参数翻译 |
| 语言持久化 | localStorage | 无需后端，刷新后记住偏好 |
| 切换方式 | 全量 `render()` | 保证一致性，避免局部遗漏 |
| Fallback | 回退到中文 | 即使翻译缺失，页面仍可读（中文为主开发语言） |
| 无外部依赖 | 零 npm 包 | 翻译系统仅 100 行，无学习成本，易于审计 |
