# 理解 URL 与短码格式

drand-draw 的每一场抽奖都由一组参数唯一定义：使用哪条 drand 链、截止到哪个 round、参与人数多少、设了几个奖等、各奖等有多少人中奖。当你需要把一场抽奖分享给粉丝或存入书签时，有两种方式传递这些参数：**完整 URL** 和 **短码**。两者在语义上等价，但形式和用途不同。

## URL 格式：直白但较长

一场抽奖的完整 URL 看起来像这样：

```
https://drand-draw.pages.dev/#/?chain=quicknet&deadline=1692803367&n=100&prizes=1,3&winners=42,15
```

`#/?` 之后的部分是 URL query 参数，五个字段的含义如下：

| 参数 | 示例值 | 含义 |
|------|--------|------|
| `chain` | `quicknet` | drand 链名称，可选 `quicknet`（3 秒一轮）、`default`（30 秒一轮）、`evmnet`（3 秒一轮） |
| `deadline` | `1692803367` | 截止 round 的 Unix 时间戳（秒），系统将取此时间戳之前的最新 round 作为抽奖依据 |
| `n` | `100` | 参与人数，即从多少个用户中抽取 |
| `prizes` | `1,3` | 各奖等的奖品数量；`1,3` 表示一等奖 1 名、二等奖 3 名 |
| `winners` | `42,15` | 各奖等的获奖人数；`42,15` 表示一等奖抽 42 人、二等奖抽 15 人 |

URL 的优点是**自解释**——任何人看到 `chain=quicknet` 就知道用的是哪条链。缺点是长度较长，在 Twitter、微博等社交平台上容易被识别为外链，触发链接限流或折叠。

[来源](src/encode.js#L41-L68)

## 短码格式：紧凑且防限流

同一场抽奖的短码版本：

```
q-66364280-2s-1,3-2a,f,4e
```

短码用连字符 `-` 分隔四个或五个段，每个段都比 URL 中的对应参数短得多。编码规则如下：

### 1. 链前缀：单字母映射

| 链名称 | 前缀 |
|--------|------|
| `quicknet` | `q` |
| `default` | `d` |
| `evmnet` | `e` |

`q` 一眼就能对应到 `quicknet`，比写全称省了 7 个字符。[来源](src/chains.js#L43-L44)

### 2. deadline：十六进制（base16）

Unix 时间戳 `1692803367` 转成十六进制是 `64e62127`。十进制转十六进制后，数字的表示长度平均缩短约 60%。[来源](src/encode.js#L6)

### 3. n：三十六进制（base36）

`100` 转成三十六进制为 `2s`（2×36 + 28 = 100）。base36 使用 `0-9a-z` 共 36 个字符，比十进制紧凑，又比纯十六进制更短。[来源](src/encode.js#L7)

### 4. prizes 与 winners：逗号分隔的 base36

奖等数量 `[1, 3]` → `1,3`（每个数字独立转 base36，1 和 3 在 base36 下仍为 `1` 和 `3`）。

获奖人数 `[42, 15]` → `2a,f,4e`（42→`2a`，15→`f`，158→`4e`）。多个值用逗号连接，没有空格。[来源](src/encode.js#L8-L13)

把上述所有段拼起来就是：

```
q-64e62127-2s-1,3-2a,f,4e
```

### 解码就是逆向操作

JavaScript 版 `decodeShortCode()` 和 Python 版 `decode_short_code()` 都遵循同样的流程：按 `-` 拆分 → 查表还原链名 → base16 解析 deadline → base36 解析 n → 逗号分割并 base36 解析 prizes 和 winners。[来源](src/encode.js#L16-L31) [来源](cli/drand_draw/encode.py#L47-L67)

## 为什么需要短码？

社交平台（微博、Twitter、Discord）对 URL 有特殊处理：自动生成卡片预览、计入链接数量、对外链进行限流或降权。短码**不含点号或斜杠**，平台不会把它识别为 URL，因此：

- 不会触发链接预览（避免爬虫抓取）
- 不会计入"含外链"的惩罚规则
- 可以在正文中直接书写，排版更干净

用户复制短码后，在 drand-draw 页面的输入框中粘贴即可还原所有参数。[来源](components/CreateDraw.js#L151-L152)

## 格式对照总览

| 特性 | URL 格式 | 短码格式 |
|------|----------|----------|
| 示例 | `#/?chain=quicknet&deadline=1692803367&n=100&prizes=1,3&winners=42,15` | `q-64e62127-2s-1,3-2a,f,4e` |
| 可读性 | 自解释，无需查表 | 需要了解编码规则 |
| 是否被识别为链接 | 是 | 否 |
| 长度（字符数） | 约 80+ | 约 25-40 |
| 典型用途 | 书签、网页内跳转 | 社交平台分享 |
