# 快速开始：5 分钟完成一次可验证抽奖

本文带你从零跑通一次完整的抽奖流程——先走网页版感受全貌，再走 CLI 版理解底层命令。你不需要了解 drand 原理，不需要写代码。

---

## 网页版：从发起抽奖到验证结果

打开 [drand-draw.pages.dev](https://drand-draw.pages.dev)，你会看到三个标签页：「发起抽奖」「验证抽奖」「指南」。我们只用前两个。

### 第一步：发起一场抽奖

点击 **「发起抽奖」** 标签，进入参数填写界面：

1. **选择链（drand 网络）** — 下拉选 `QuickNet`（推荐）。它是 3 秒一轮的 drand 网络，你设的截止时间会对应到一个精确的 Round 编号。旁边的小字显示它的方案名和出块周期。 [来源](src/chains.js#L3-L18)

2. **参与人数 N** — 填你预期的参与总数，比如 `100`。说明文档见 [安全边界与参数限制](安全边界与参数限制.md)。

3. **截止时间** — 选择日期和时间，再选你的 **时区**。系统会把本地时间自动转为 Unix 时间戳发送。**必须选未来的时间**，否则无法开奖。

4. **奖项** — 默认「一等奖 1 人」。点击底部的 **「+ 添加奖项」** 可以增设二等奖、三等奖，每个奖项的人数都可以单独填。所有奖项人数之和不能超过 N。 [来源](src/components/CreateDraw.js#L108-L122)

5. 点击 **「生成抽奖链接」**。

你会看到两个输出：

| 输出 | 示例 | 用途 |
|------|------|------|
| 🔗 **链接** | `drand-draw.pages.dev/#/?chain=quicknet&deadline=1715000000&n=100&prizes=1` | 直接发在社交平台，粉丝点开即用 |
| 🔢 **短码** | `q-66364280-2s-1` | 怕链接被平台限流？发短码让粉丝粘贴 |

点击旁边的复制按钮一键复制。详情见 [理解 URL 与短码格式](理解-url-与短码格式.md)。 [来源](src/components/CreateDraw.js#L142-L166)

将链接（或短码）公开发布在你的抽奖推文中。**必须在截止时间之前发布**，否则粉丝无法确认截止时间是否被篡改。

### 第二步：等待截止

粉丝打开链接会看到 **倒计时页面**。页面显示剩余时间、截止时间、以及预期开奖的 Round 编号——这是根据 `deadline`、`genesisTime` 和 `period` 计算出来的。 [来源](src/lottery.js#L1-L6)

此时粉丝可以放心：截止时间已经在链接里写死，博主无法在后台修改。 [来源](src/components/DrawStatus.js#L64-L99)

### 第三步：开奖

截止时间一到，页面自动变成 **「已过期」** 状态，并出现 **「开奖」** 按钮。博主（或任何人）点击该按钮，系统会：

1. 连接 drand 网络，获取对应 Round 的真随机数 `randomness`
2. 用该随机数计算出中奖编号
3. 把结果写入 URL，生成新的带中奖结果的链接 [来源](src/components/DrawStatus.js#L105-L180)

开奖结果页面会显示：

```
Round: #7398878
Randomness: a3f25c8d...
Prize Tier 1 (1 winner): #42
```

并附带**复制链接**、**复制短码**、**复制分享文本**三个按钮。 [来源](src/components/DrawStatus.js#L181-L194)

### 第四步：验证

粉丝（或任何第三方）拿到结果链接或短码后，在网页任意时刻打开，页面会自动：

1. 从 drand 网络获取同一 Round 的随机数
2. 用相同算法重新计算中奖编号
3. 与博主公布的结果比对 ✅ / ❌ [来源](src/components/DrawStatus.js#L227-L273)

验证通过显示绿色 **「验证通过」** 标记，验证失败显示红色 **「验证失败」** 并列出实际计算出的中奖编号供对比。

你也可以在首页 **「验证抽奖」** 标签下，**手动输入**链名、Round、N、奖项和中奖编号来独立验证，不依赖他人提供的任何链接。 [来源](src/components/DrawStatus.js#L277-L348)

> 更详细的博主/粉丝视角请参阅 [博主：发起一次抽奖](博主-发起一次抽奖.md) 和 [粉丝：验证抽奖结果](粉丝-验证抽奖结果.md)。

## CLI 版：三条命令跑通全程

网页版适合博主快速发起，CLI 版则更适合**验证阶段**——你可以完全脱离浏览器，在终端里一条命令验真伪。

### 安装

确保 Python ≥ 3.10，然后：

```bash
cd cli
pip install .
```

安装后验证：

```bash
drand-draw --help
```

你应该看到 `verify`、`compute`、`encode`、`decode` 四条子命令。 [来源](cli/drand_draw/__main__.py#L68-L96)

不安装也可以直接运行：`python -m drand_draw <command>`。 [来源](cli/README.md#L10-L12)

### `compute` — 从截止时间和参与人数计算开奖结果

假设你在 QuickNet 上设了截止时间 `1715000000`（Unix 秒），参与人数 100，两个奖项分别 1 人和 3 人：

```bash
drand-draw compute --chain quicknet --deadline 1715000000 --n 100 --prizes 1,3
```

输出示例：

```
Deadline: 2026-05-06 12:53:20 UTC
Round: #7398878
Randomness: a3f25c8d1e7b...
Prize Tier 1 (1 winner): #42
Prize Tier 2 (3 winners): #15, #78, #43
```

命令内部逻辑：用 `deadline` 算出 Round → 从 drand 网络获取该 Round 的随机数 → 用 `compute_winners` 算法算出中奖编号。 [来源](cli/drand_draw/lottery.py#L1-L28)、[来源](cli/drand_draw/__main__.py#L43-L62)

> **⚠️ 注意：`--deadline` 必须使用 Unix 时间戳（秒）**，不是人类可读的日期。网页版会自动转换，CLI 版需要自己转换。可用 `date +%s`（Linux/macOS）或在线转换工具获取。

### `encode` — 将参数编码为短码

方便在社交平台分享，避免长链接被限流：

```bash
drand-draw encode --chain quicknet --deadline 1715000000 --n 100 --prizes 1,3
```

输出：`q-66364280-2s-1,3`

如果已经开奖，也可以把中奖编号一起编码：

```bash
drand-draw encode --chain quicknet --deadline 1715000000 --n 100 --prizes 1,3 --winners 42,15,78,43
```

短码格式：`{链前缀}-{deadline_hex}-{N_base36}-{奖项}-{中奖编号}`。`q` 表示 quicknet，`66364280` 是 `1715000000` 的十六进制，`2s` 是 100 的 base36。 [来源](src/encode.js#L1-L15)、[来源](src/chains.js#L39-L42)

解码验证：

```bash
drand-draw decode "q-66364280-2s-1,3-2a,f,4e,2b"
```

输出：

```
Chain: quicknet
Deadline: 1715000000 (2026-05-06 12:53:20 UTC)
N: 100
Prizes: 1, 3
Winners: #42, #15, #78, #43
```

[来源](cli/drand_draw/__main__.py#L99-L113)

### `verify` — 验证博主宣称的中奖结果是否真实

这是 CLI 最核心的用途。假设博主声称中奖编号是 `42, 15, 78, 43`：

```bash
drand-draw verify --chain quicknet --round 7398878 --n 100 --prizes 1,3 --winners 42,15,78,43
```

输出示例：

```
Round: #7398878
Randomness: a3f25c8d1e7b...
Prize Tier 1 (1 winner): #42  ✅
Prize Tier 2 (3 winners): #15, #78, #43  ✅
Verification: PASSED
```

如有作弊，输出会显示 ❌ 并给出**实际应得的中奖编号**。 [来源](cli/drand_draw/__main__.py#L13-L42)

`verify` 不依赖 deadline，只需要 Round——这意味着即使博主的原始推文被删除，只要 Round 编号还在，任何人都可以终身验证。
