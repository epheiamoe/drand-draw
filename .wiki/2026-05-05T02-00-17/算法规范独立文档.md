# 算法规范独立文档

drand-draw 的核心价值在于**可独立验证**。本文档从完整指南中提取算法规范，目标是让任何开发者——不使用本工具的任何代码——仅凭本节描述的规范，就能用自己选择的编程语言完全复现抽奖结果。

整个抽奖流程是一条从"参与列表"到"中奖编号"的**确定性计算链**：给定相同的输入（参与者、截止时间、drand 链），任何人都必须得到完全相同的输出。下文逐节拆解这条计算链的每一个环节。

## 1. 参与列表：去重 → ASCII 排序 → 编号

所有抽奖结果的计算起点，是一个**确定的有序编号列表**。

### 变换规则

```
原始输入:  ["@Alice", "@bob", "@Alice", "@Charlie"]
                                  ↓ 去重
           ["@Alice", "@bob", "@Charlie"]
                                  ↓ ASCII 升序排序
           ["@Alice", "@Charlie", "@bob"]
                                  ↓ 从 0 编号
           0: "@Alice", 1: "@Charlie", 2: "@bob"
```

三个步骤缺一不可：

1. **去重**：移除重复的用户名。同一用户多次互动只计一次，由博主诚实处理。
2. **排序**：按 ASCII 码升序排列。字符优先级：`"0" < "9" < "@" < "A" < "Z" < "_" < "a" < "z"`。JavaScript 实现使用 `String.prototype.localeCompare` 配合 `'en'` locale 和 `sensitivity: 'base'` 参数。Python 实现直接用 `sorted()` 即按 Unicode 码点排序，对 ASCII 范围内的字符等价于 ASCII 排序。
3. **编号**：从 0 开始连续分配。有效参与总人数记为 `N`。

### 设计意图

编号决定了中奖者的身份。0 号对应排在第一位的参与者，编号与中奖编号直接对应。**排序规则必须硬编码**，不可由博主配置——这是防作弊的基础之一。

## 2. Round 计算：将 deadline 映射到 drand 网络中的轮次

drand 网络按固定**周期 (period)** 产生随机数，每周期一轮，从创世时间开始计数。Round 计算是将抽奖截止时间（Unix 时间戳）映射到具体某一轮次的数学公式。

### 公式

```
round = floor((deadline - genesis_ts) / period) + 1
```

其中 `genesis_ts` 是链的创世时间戳，`period` 是出块周期（秒）。`+1` 是因为 drand 的轮次从 1 开始计数。

### 三条链的参数表

| 链 | `chain_hash` (64 字符十六进制) | `genesis_ts` | `period` | 适合场景 |
|---|---|---|---|---|
| **quicknet** | `52db9ba70e0cc0f6eaf7803dd07447a1f5477735fd3f661792ba94600c84e971` | 1692803367 | 3s | 需要快速结果的短时抽奖 |
| **default** | `8990e7a9aaed2ffed73dbd7092123d6f289930540d765133172e51b2ce` | 1595431050 | 30s | 通用场景 |
| **evmnet** | `04f1e9062b8a81f848fded9c12306733282b2727ecced50032187751166ec8c3` | 1727521075 | 3s | 面向 EVM 兼容链的场景 |

JavaScript 实现中，如果 `elapsed < 0`（deadline 早于创世时间），返回 `-1` 表示无效。

## 3. 获取随机数：HTTP API 与响应格式

drand 网络公开 HTTP API，任何人都可以通过标准的 REST 请求获取指定轮次的随机数。

### 请求

```
GET https://api.drand.sh/{chain_hash}/public/{round}
```

### 响应（JSON）

```json
{
  "round": 7398878,
  "randomness": "a3f25c8d1e7b...",
  "signature": "92daf574..."
}
```

其中 `randomness` 字段是 **64 字符的十六进制字符串**（32 字节），用作后续中奖计算的熵源。`signature` 可用于对 `randomness` 进行 BLS 签名验证（验证这把随机数是否由 drand 网络合法产生）。

### 备用 Relay 节点

当主节点不可用时，可以使用以下备用地址。JavaScript 实现中内置了重试与超时机制（8 秒超时），依次尝试各 relay。

- `https://api.drand.sh`
- `https://api2.drand.sh`
- `https://drand.cloudflare.com`

## 4. 中奖计算：核心算法

这是整个系统的核心。**每一个奖位**都由 `randomness` 的一个**唯一分片**通过取模运算独立确定。

### 完整算法（伪代码）

```
function computeWinners(randomness, N, prizeTiers):
  if sum(prizeTiers) > N:
    throw Error("总奖位数超过参与人数 N")

  winners = []       // 中奖编号列表
  used = {}          // 已占用的编号集合

  for i in 0..len(prizeTiers)-1:        // 遍历奖项层级
    for j in 0..prizeTiers[i]-1:        // 遍历该层每个奖位
      shift = len(winners)              // 关键：当前已确定的中奖人数
      seedHex = randomness + toHex(shift)   // 拼接，无 0x 前缀
      bigVal = BigInt('0x' + seedHex)
      idx = bigVal % N

      // 碰撞顺延（环形查找）
      attempts = 0
      while idx in used:
        idx = (idx + 1) % N
        attempts++
        if attempts >= N:
          throw Error("Collision resolution failed")

      used.add(idx)
      winners.append(idx)

  return winners
```

### shift 的作用：熵分片

`shift` 是算法中最重要的设计决策。它的值等于当前已确定的中奖人数（即 `winners.length`）。每个奖位使用 `randomness + toHex(shift)` 作为其专属种子：

| 奖位序号 | shift | 种子拼接 | 说明 |
|---|---|---|---|
| 第 1 个 | 0 | `randomness` + `"0"` | 直接使用 randomness 末尾补 0 |
| 第 2 个 | 1 | `randomness` + `"1"` | 末尾补 1 |
| 第 3 个 | 2 | `randomness` + `"2"` | 末尾补 2 |
| ... | ... | ... | ... |
| 第 10 个 | 9 | `randomness` + `"9"` | 补 9 |
| 第 11 个 | 10 | `randomness` + `"a"` | 补 a（十六进制） |
| ... | ... | ... | ... |

**为什么不用 `i + j` 而用 `shift`？** 因为 `shift` 是当前已确定的中奖人数，而非简单的循环变量。如果某个奖位因碰撞顺延而改变了后续奖位的 `shift` 值……不对，`shift` 在顺延过程中**不会改变**，它是在每一轮外循环开始时根据 `winners.length` 确定的。这种方式保证了：**即使发生碰撞顺延，后续奖位使用的 shift 也不会被影响**。

更精确地说，`shift` 在每次内循环开始时捕获 `winners.length` 的当前值。由于 `winners` 每确定一个奖位就追加一个元素，这个值恰好对每个奖位都是唯一的——它就是该奖位在全局中奖序列中的序号（从 0 开始）。

### 碰撞顺延逻辑

当 `idx = bigVal % N` 计算出的编号已经被占用时，算法采用**环形线性探测**：

```
idx = (idx + 1) % N
```

即简单地加 1 并取模，形成一个环形查找。如果尝试了 `N` 次仍未找到空位，抛出 "Collision resolution failed" 异常——但这在数学上不可能发生，因为 `totalPrizes <= N` 保证了至少有一个空位。

## 5. 短码编码格式

短码是抽奖参数的紧凑表示，用于在 URL 中分享。格式如下：

```
{chain}-{deadline_hex}-{N_base36}-{prizes}-{winners}
```

### 各字段说明

| 字段 | 编码方式 | 示例 |
|---|---|---|
| `chain` | 单字符前缀：`q`=quicknet, `d`=default, `e`=evmnet | `q` |
| `deadline` | 小端十六进制，不含 `0x` 前缀 | `66364280` |
| `N` | base36 编码（数字 0-9，小写字母 a-z） | `2s` |
| `prizes` | 逗号分隔的 base36 数字，可选 | `1,3` |
| `winners` | 逗号分隔的 base36 数字，可选 | `2a,f,4e` |

### 示例

```
q-66364280-2s-1,3                      仅配置（不含结果）
q-66364280-2s-1,3-2a,f,4e            含中奖编号的全量信息
```

base36 是一种紧凑的数字编码，使用 `0-9a-z` 共 36 个字符。例如，参与人数 `N=100` 编码为 `2s`（仅 2 字符），`N=1000000` 编码为 `lfls`。JavaScript 使用 `Number.prototype.toString(36)`，Python 使用 `int(s, 36)` 进行互转。

短码的编解码逻辑在 JavaScript 和 Python 实现中保持一致，编码结果可跨语言解析。

## 6. 验证步骤：全流程串联

将上述所有环节串联，即可完成一次完整的独立验证。以下是逐步骤说明：

### 步骤 1：确认基本参数

从短码或 URL 中提取：
- `deadline`：抽奖截止的 Unix 时间戳
- `N`：有效参与人数
- `chain`：使用的 drand 链标识
- `prizeTiers`：奖项层级配置

### 步骤 2：计算 Round

```
round = floor((deadline - genesis_ts) / period) + 1
```

使用对应链的 `genesis_ts` 和 `period`（见第 2 节的参数表）。

### 步骤 3：获取 Randomness

```bash
# 使用 curl 从 drand 网络获取
curl https://api.drand.sh/{chain_hash}/public/{round}

# 提取 randomness 字段（64 字符十六进制字符串）
```

备用 relay：`api2.drand.sh`、`drand.cloudflare.com`。

### 步骤 4：计算中奖编号

调用 `computeWinners(randomness, N, prizeTiers)` 函数（算法见第 4 节）。

### 步骤 5：比对结果

将计算出的中奖编号列表与博主公示的编号逐一比对。编号对应排序后的参与者列表中相应序号的用户。

### 命令行验证完整示例

```bash
# 1. 定义参数
CHAIN_HASH="52db9ba70e0cc0f6eaf7803dd07447a1f5477735fd3f661792ba94600c84e971"
ROUND=7398878
N=100

# 2. 获取 randomness
RANDOMNESS=$(curl -s "https://api.drand.sh/${CHAIN_HASH}/public/${ROUND}" \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['randomness'])")

# 3. 用 Python 计算中奖编号
python3 -c "
r = '$RANDOMNESS'
N = $N
prizes = [1, 3]
winners = []
used = set()
for i, c in enumerate(prizes):
  for j in range(c):
    s = len(winners)
    idx = int(r + format(s, 'x'), 16) % N
    while idx in used:
      idx = (idx + 1) % N
    used.add(idx)
    winners.append(idx)
print('Winners:', winners)
"
```

这段代码**不依赖本工具的任何模块**，只使用 Python 标准库，可作为独立验证的最小可靠示例。

## 7. 安全边界

算法规范中明确定义了各参数的上限，超出这些上限可能导致计算错误或安全风险：

| 参数 | 最大值 | 技术原因 |
|---|---|---|
| **N**（参与人数） | 1,048,575 | 约 100 万，保证计算性能与编号表示简洁 |
| **奖项层级** | 15 层 | 兼顾大多数抽奖场景，防止过度复杂的配置 |
| **每层人数** | 1,000 | 避免单层奖位过多导致碰撞概率剧增 |
| **deadline** | 2106 年 | Unix 32 位时间戳上限 |
| **用户名长度** | 100 字符 | 保证 Canvas 导出列表中文本不截断 |

这些边界在 JavaScript 前端（CreateDraw 组件）和 CLI 中均有校验。

## 总结：独立复现的条件

任何一个第三方开发者，只要具备以下三样东西，就可以不依赖本工具、用任何语言完整复现抽奖结果：

1. **抽奖参数**（deadline、N、chain、prizeTiers）—— 从短码或 URL 中提取
2. **参与者排序规则**（去重 → ASCII 排序 → 从 0 编号）—— 第 1 节
3. **drand 随机数**（通过 HTTP API 获取）—— 第 2-3 节

然后执行第 4 节的中奖算法，比对第 6 节的验证流程。整个过程的**确定性**保证了：只要输入一致，输出必然一致。这让博主无法篡改结果，也让粉丝无需信任任何人——只需信任数学。
