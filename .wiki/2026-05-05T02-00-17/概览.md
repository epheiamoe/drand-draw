# drand-draw 概览

## 这是什么？

**drand-draw** 是一个基于 [drand](https://drand.love) 去中心化随机数网络的社交平台抽奖工具。博主设定截止时间 → 粉丝参与 → 截止后由 drand 网络产出的公开随机数决定中奖者 → 结果任何人可独立验证。

**这不是 Web3/NFT 项目，不运行在区块链上，不存储任何数据。** 整个应用是纯静态页面，部署在 Cloudflare Pages，没有后台服务器，没有数据库。抽奖的每一环——从 Round 计算到中奖编号生成——都在浏览器或本地 CLI 中完成，drand 只提供不可篡改的随机数源。

## 核心特性

| 特性 | 说明 |
|------|------|
| **零后台** | 纯静态网站 + Python CLI，无服务器、无数据库、无状态 |
| **可验证** | 抽奖全流程透明，粉丝可自行验证每一步是否公正 |
| **中英双语** | 自动检测浏览器语言，支持手动切换，详见[国际化实现](国际化实现.md) |
| **多链支持** | 接入 quicknet（3秒一轮）、default（30秒一轮）、evmnet（3秒一轮）三条 drand 网络 |
| **多奖项** | 支持一/二/三等奖等多层级配置，每个奖项独立摇号 |
| **短码分享** | URL 和短码两种分享方式，短码可规避社交平台外链限流 |

## 架构一览

整个系统只有一个部署单元：**Cloudflare Pages**。

```
┌────────────────────────────────────────────────┐
│                  Cloudflare Pages               │
├────────────────────────────────────────────────┤
│  src/main.js          ← 入口 + Hash 路由        │
│  src/chains.js        ← 三条链的元数据配置       │
│  src/i18n.js          ← 中英双语翻译系统         │
│  src/encode.js        ← 短码 ⇄ 编解码           │
│  src/api.js           ← drand HTTP API 封装     │
│  src/lottery.js       ← 抽奖核心算法             │
│  src/components/                                │
│   ├── CreateDraw.js   ← 发起抽奖表单             │
│   ├── DrawStatus.js   ← 开奖/验证页面三态展示    │
│   └── Guide.js        ← 使用说明/规则/原理       │
└────────────────────────────────────────────────┘
```

各模块职责清晰：`chains.js` 定义三条链的 genesis time、period 和 relay 端点；`lottery.js` 封装 Round 计算和中奖算法，与 Python CLI 版完全一致；`api.js` 负责从 drand relay 拉取 randomness，含多 Relay 容错；`encode.js` 实现 URL/短码的双向转换。页面路由由 `main.js` 基于 Hash 管理，详见 [SPA 路由与页面三态](spa-路由与页面三态.md)。

## 防作弊机制：攻击面分析

抽奖最大的信任问题是：**博主能否操纵结果？** 系统从算法层面封堵了六种常见攻击路径：

| 攻击方式 | 防御机制 |
|----------|----------|
| 博主在看到 randomness 后挑选有利的 round | Round 由 deadline 唯一确定，deadline 在 URL/短码中锁定，不可更改 |
| 博主对不同的验证者谎报不同的 N | N 编码在 URL/短码中，页面直接解码显示，不可否认 |
| 博主使用不同的排序规则 | 排序规则由页面硬编码声明（ASCII 升序），博主无法否认 |
| 博主篡改 drand 返回的随机数 | drand 返回结果由 League of Entropy 多节点签名，可用公钥验证 |
| 博主不断刷新直到出现满意的结果 | 同一个 URL/短码只能产出一个 winner 结果，粉丝可自行验证 |
| 博主操控参与者列表 | 候选名单需博主在 Phase 2 公开，粉丝需自行核对（社会约束） |

前五条由算法和密码学保证，最后一条仍需社会监督：博主需在 deadline 前公开 URL/短码，并在开奖后公布完整候选列表。详见[抽奖的安全模型](抽奖的安全模型.md)。
