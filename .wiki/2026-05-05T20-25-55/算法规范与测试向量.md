### 1.1 参与列表排序与编号

**输入**: 参与者用户名列表（字符串数组）  
**输出**: 编号 0 到 N-1 的有序列表  

```
步骤:
  1. 去重：移除重复的用户名
  2. 排序：按字符串的 ASCII 码升序排列（逐字节比较 UTF-8 编码）

### 1.2 Round 计算

Round 是 drand 网络中的轮次编号，由截止时间（deadline）和链参数唯一确定：

```
round = floor((deadline - genesis_ts) / period) + 1
```

三条支持的链参数：

| 链 | chain_hash | genesis_ts | period(秒) |
|----|-----------|-----------|-----------|
| quicknet | `52db9ba7...c84e971` | 1692803367 | 3 |
| default | `8990e7a9...51b2ce` | 1595431050 | 30 |
| evmnet | `04f1e906...66ec8c3` | 1727521075 | 3 |

**JavaScript 实现** (`src/lottery.js`):

```js
export function computeRound(chainId, deadlineUnix) {
  const chain = CHAINS[chainId]
  const elapsed = deadlineUnix - chain.genesisTime
  if (elapsed < 0) return -1
  return Math.floor(elapsed / chain.period) + 1
}
```

**Python 实现** (`cli/drand_draw/lottery.py`):

```python
def compute_round(deadline, genesis, period):
    elapsed = deadline - genesis
    if elapsed < 0:
        return -1
    return math.floor(elapsed / period) + 1
```

### 1.3 种子派生函数

这是整个系统的核心密码学操作。每个奖位使用一个独立的 SHA-256 哈希派生种子：

```
seedHex = SHA256(randomness + ':' + shift)
```

其中：
- `randomness`：64 字符十六进制字符串，来自 drand 网络的输出
- `shift`：当前已产生的奖位数（从 0 开始递增）
- `+` 表示字符串拼接（不是字节拼接）
- `:` 是 ASCII 冒号字符（0x3A）

**JavaScript 实现** (`src/lottery.js#L9-L16`):

```js
async function deriveSeed(randomness, shift) {
  const input = randomness + ':' + shift
  const enc = new TextEncoder().encode(input)
  const hash = await crypto.subtle.digest('SHA-256', enc)
  const hex = Array.from(new Uint8Array(hash))
    .map(b => b.toString(16).padStart(2, '0')).join('')
  return hex
}
```

**Python 实现** (`cli/drand_draw/lottery.py#L5-L8`):

```python
def _derive_seed(randomness, shift):
    data = f'{randomness}:{shift}'.encode()
    return hashlib.sha256(data).hexdigest()
```

> **关键语义**: 输入字符串是 `randomness + ":" + str(shift)`，而不是二进制拼接。这意味着 randomness 的十六进制字符作为 ASCII 文本传递，而非作为原始字节。两种语言必须使用完全相同的字符串输入。

### 1.4 中奖计算

```
function computeWinners(randomness, N, prizeTiers):
  if sum(prizeTiers) > N: throw Error
  winners = []
  used = {}

  for each prize tier i:
    for each slot j in tier i:
      shift = len(winners)
      seedHex = SHA256(randomness + ':' + shift)
      bigVal = BigInt('0x' + seedHex)
      idx = bigVal % N

      while idx in used:         // 碰撞处理
        idx = (idx + 1) % N      // 环形顺延

      used.add(idx)
      winners.append(idx)

  return winners
```

碰撞处理采用**线性探针（Linear Probing）**策略：当哈希取模后的编号已被占用时，递增 1（模 N），直到找到空闲位置。最坏情况下，如果所有 N 个位置均被占用（即 `totalPrizes == N` 且哈希恰好构成一个循环排列），算法会在 N 次尝试后抛出异常。

**JavaScript 实现** (`src/lottery.js#L18-L42`):

```js
export async function computeWinners(randomness, n, prizeTiers) {
  const totalPrizes = prizeTiers.reduce((a, b) => a + b, 0)
  if (totalPrizes > n) {
    throw new Error(`Total prizes (${totalPrizes}) exceeds N (${n})`)
  }
  const winners = []
  const used = new Set()
  for (let i = 0; i < prizeTiers.length; i++) {
    for (let j = 0; j < prizeTiers[i]; j++) {
      const shift = winners.length
      const seedHex = await deriveSeed(randomness, shift)
      const bigVal = BigInt('0x' + seedHex)
      let idx = Number(bigVal % BigInt(n))
      let attempts = 0
      while (used.has(idx)) {
        idx = (idx + 1) % n
        attempts++
        if (attempts >= n) throw new Error('Collision resolution failed')
      }
      used.add(idx)
      winners.push(idx)
    }
  }
  return winners
}
```

## 5. 跨平台 SHA-256 一致性

### 5.1 三种实现必须产生相同结果

| 平台 | API | 返回类型 | SHA-256 一致性 |
|------|-----|---------|--------------|
| Web (Browser) | `crypto.subtle.digest('SHA-256', ...)` | Promise\<ArrayBuffer\> | ✅ FIPS 180-4 标准 |
| Node.js | `crypto.createHash('sha256').update(...)` | 同步返回 Hex | ✅ 同上 |
| Python | `hashlib.sha256(data).hexdigest()` | 返回 Hex 字符串 | ✅ 同上 |

### 5.2 常见陷阱

**陷阱 1: 输入编码不一致**

JavaScrpt 端：
```js
const input = randomness + ':' + shift
const enc = new TextEncoder().encode(input)
// TextEncoder 始终输出 UTF-8 字节
```

Python 端：
```python
data = f'{randomness}:{shift}'.encode()
# .encode() 默认 UTF-8，与 TextEncoder 一致
```

由于 randomness 全是 ASCII 十六进制字符（`0-9a-f`），`:` 也是 ASCII，`shift` 是数字字符串，因此 UTF-8 编码与 ASCII 编码完全等价。输入字符串相同则 SHA-256 输出必然相同。

**陷阱 2: 大整数精度**

JavaScript 的 `BigInt` 可以处理任意精度的十六进制数（`0x` + 64 字符 hex = 256 位）。Python 的 `int(seed_hex, 16)` 同样支持任意精度。取模运算 `bigVal % N` 在两种语言中语义一致。

**陷阱 3: 碰撞处理顺序**

测试 `used.has(idx)` 时必须使用**先判再用**的顺序——先检查当前 idx 是否被占用，若占用则递增。两种语言的实现均遵循同一顺序，碰撞链相同。

### 5.3 一致性断言

对于任意相同的 `(randomness, N, prizeTiers)` 输入：

```
Python  compute_winners(r, N, p) == JS  computeWinners(r, N, p)
```

这两个调用返回的 winners 数组元素顺序和数值必须完全相同。
