# 概览

**drand-draw** 是一个零后端、纯前端的社交平台抽奖工具。它不设数据库、不跑服务器、不依赖区块链——整个应用就是一个静态网页（托管在 Cloudflare Pages）加一个 Python 命令行工具。所有抽奖逻辑由 [drand](https://drand.love) 去中心化随机信标网络的公开随机数驱动：博主设定截止时间 → 粉丝参与 → 截止后用 drand 的随机数开奖 → 任何人可独立验证结果。**整个过程不需要信任任何中介，也不存储任何用户数据。** [来源](README.md#L5-L7)

## 核心特性

**无后端存储。** 所有抽奖参数（链、截止时间、参与人数 N、奖项配置、中奖编号）都编码在 URL 或短码中，不落服务器。博主分享一个链接，就等于公开了全部参数。粉丝拿到链接，页面自动解码并验证，不需要任何注册或登录。 [来源](PLAN.md#L14-L15)

**多链支持。** 集成三条 drand 网络：QuickNet（3 秒一轮，适合实时开奖）、Default（30 秒一轮，经典网络）、EVMNet（3 秒一轮，BN254 曲线）。三条链的随机数均由 League of Entropy 多方签名生成，任何人无法预测或操纵。博主可在创建抽奖时自由选择。 [来源](src/chains.js#L1-L47)

**短码分享。** 为了绕过社交平台的外链限流、方便粉丝在评论区直接复制，系统支持将抽奖参数压缩为短码格式。短码形如 `q-66364280-2s-1,3`，包含链标识、截止时间、参与人数、奖项配置，甚至可包含中奖编号。 [来源](PLAN.md#L107-L119)

> 详细机制见 [短码分享机制](短码分享机制.md)。

## Web 版 vs CLI 版

两个版本共享同一套核心算法（JavaScript 与 Python 实现完全一致），但使用场景和能力各有侧重。

| 能力维度 | Web 版 | CLI 版 |
|----------|--------|--------|
| **访问方式** | 浏览器打开链接即用，零安装 | 需 Python 3.10+，`pip install` 安装 |
| **创建抽奖** | 可视化表单，填写参数即生成链接/短码 | 不支持（CLI 不包含创建表单） |
| **验证开奖** | 粘贴链接或短码，页面自动解析、调 drand API 验证 | `drand-draw verify` 命令手动指定参数验证 |
| **计算中奖** | 自动触发，开奖后立即展示结果 | `drand-draw compute` 离线计算，适合脚本批量处理 |
| **编码/解码** | URL 与短码自动互转 | `drand-draw encode` / `drand-draw decode` 手动转换 |
| **截止时间** | 输入框选日期时间 | `drand-draw deadline` 工具函数，手动算 round |
| **适用场景** | 博主发起抽奖、粉丝一键验证 | 高级用户离线验证、集成到自动化流程 |

[来源](README.md#L24-L31) | [来源](cli/drand_draw/__main__.py#L1-L30)

> Web 版开箱即用指南见 [快速开始](快速开始.md)，CLI 版安装与命令详解见 [CLI 工具安装与基础用法](cli-工具安装与基础用法.md)。

## 项目定位声明

项目 README 开篇即强调：

> **这不是 Web3/NFT 项目，不运行在区块链上，不存储任何数据。** [来源](README.md#L7)

中文完整指南中也有相同的声明：

> **这不是区块链项目，不是 Web3 工具，不是 NFT mint，不是企业级随机数服务。** 这是一个给 X/Twitter 博主用的社交平台抽奖工具，仅此而已。 [来源](public/GUIDE.md#L3)

这一声明定义了项目的边界：它**借用**了去中心化随机信标（drand）的公开随机数来保证抽奖的透明与公正，但本身**不是**一个区块链应用。没有智能合约、没有 Token、没有链上交易。它就是一个纯粹的抽奖小工具——只不过随机数的来源足够可信。

## 信任模型

整个系统的信任建立在两个公开承诺之上：

1. **Round 由 deadline 唯一锁定。** 截止时间一旦确定，对应的 drand round 号就固定了，博主无法在知道随机数后反悔换 round。
2. **参数锁定在链接/短码中。** N、奖项、中奖编号在分享时就已公开，博主无法对不同的验证者给出不同的结果。

> 完整的攻防分析见 [攻击面分析与信任模型](攻击面分析与信任模型.md)。
