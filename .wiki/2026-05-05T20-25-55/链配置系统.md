# 链配置系统

**drand（去中心化随机信标）** 是一个由多方运行的公共随机性服务网络。但"drand 随机数"并非只有一种——drand 网络实际维护着多条独立运行的 **链（chain）**，每条链有着不同的加密方案和出块节奏。本系统支持其中三条，并在 `chains.js` 中以统一结构定义。

## 三条链的完整配置

所有链配置集中定义在 [`src/chains.js`](../src/chains.js#L1-L47) 中，导出 `CHAINS` 对象。核心参数包括：链的唯一标识 `chainHash`（用于 API 路由寻址）、创世时间 `genesisTime`（Unix 秒）、出块周期 `period`（秒）、签名方案 `scheme`、以及一组 relay 节点 URL（用于故障切换）。

| 参数 | QuickNet | Default | EVMNet |
|---|---|---|---|
| **id** | `quicknet` | `default` | `evmnet` |
| **chainHash** | `52db9ba70e0cc0f6eaf7803dd07447a1f5477735fd3f661792ba94600c84e971` | `8990e7a9aaed2ffed73dbd7092123d6f289930540d7651336225dc172e51b2ce` | `04f1e9062b8a81f848fded9c12306733282b2727ecced50032187751166ec8c3` |
| **genesisTime** | `1692803367`（2023-08-23） | `1595431050`（2020-07-23） | `1727521075`（2024-09-28） |
| **period** | **3 秒** | **30 秒** | **3 秒** |
| **scheme** | `bls-unchained-g1-rfc9380` | `pedersen-bls-chained` | `bls-bn254-unchained-on-g1` |
| **publicKey**（长度） | 96 字节 | 48 字节 | 128 字节 |
| **relays** | 3 个（同下） | 3 个（同下） | 3 个（同下） |

三条链共享 **同一组 relay 节点**：

- `https://api.drand.sh`
- `https://api2.drand.sh`
- `https://drand.cloudflare.com`

## 关键差异解析

三条链最显著的差异体现在周期和加密方案上。

### period：3 秒 vs 30 秒 vs 3 秒

**出块周期**直接决定了一次抽奖需要等待的最长时间。系统通过 `computeRound` 函数将用户设定的截止时间（Unix 时间戳）映射为 drand 网络的 **round 编号**：

```js
const elapsed = deadlineUnix - chain.genesisTime
return Math.floor(elapsed / chain.period) + 1
```

[来源](../src/lottery.js#L3-L6)

这意味着：**截止时间越近、period 越小，Round 锁定越精准**。Default 链的 30 秒周期意味着用户设定的截止时间会被"坍缩"到最近的上一个 30 秒边界，不确定性窗口大。QuickNet 的 3 秒周期则将这个窗口压缩到 3 秒以内。

### scheme：三种签名算法

| 链 | scheme | 含义 | 链式？ |
|---|---|---|---|
| QuickNet | `bls-unchained-g1-rfc9380` | BLS 签名，G1 曲线，非链式，RFC9380 哈希 | 否 |
| Default | `pedersen-bls-chained` | Pedersen 参数化的 BLS，**链式**模式 | **是** |
| EVMNet | `bls-bn254-unchained-on-g1` | BLS 签名，BN254 曲线，非链式 | 否 |

**链式（chained）** 指每个 round 的随机数不仅包含信标签名，还依赖上一个 round 的输出。非链式（unchained）则每个 round 独立生成，可并行验证。EVMNet 使用 BN254 曲线（以太坊友好），QuickNet 使用 BLS12-381 曲线（更高的安全级别）。

[来源](../src/chains.js#L7-L8)

### 为什么推荐 QuickNet

在创建抽奖的 UI 中，QuickNet 被设为默认选项。原因有三：

1. **出块快**——period 仅 3 秒，从截止到出结果的等待时间最短。
2. **非链式，验证简单**——每个 round 独立，无需追溯前序 round 来验证正确性。
3. **广泛可用**——三条 relay 节点均支持，无单点故障风险。

在创建抽奖页面，用户选择不同链时，界面会动态显示该链的方案、周期和创世日期，帮助用户做决策。选择器展示信息如 `bls-unchained-g1-rfc9380 · 3s/round · genesis 2023-08-23`。

[来源](../src/components/CreateDraw.js#L98-L102)

## CHAIN\_SHORT / SHORT\_CHAIN 映射

短码分享机制中，链 ID 需要压缩为单字符前缀。`chains.js` 定义了两张双向映射表：

```js
export const CHAIN_SHORT = { q: 'quicknet', d: 'default', e: 'evmnet' }
export const SHORT_CHAIN = { quicknet: 'q', default: 'd', evmnet: 'e' }
```

[来源](../src/chains.js#L49-L50)

- `SHORT_CHAIN` 用于 **编码**（encode）：`encodeShortCode` 将链 ID 映射为单字母前缀

  ```js
  const parts = [SHORT_CHAIN[params.chain] || 'q']
  ```

- `CHAIN_SHORT` 用于 **解码**（decode）：`decodeShortCode` 将单字母前缀还原为链 ID

  ```js
  const chainId = CHAIN_SHORT[parts[0]] || 'quicknet'
  ```

[来源](../src/encode.js#L3-L4)

这样，一个完整短码的结构如 `q-672a0f0-64` 中，首字符 `q` 即代表 QuickNet 链。详细短码规范参见 [短码编解码规范](短码编解码规范.md)。

## 链配置在系统中的流转

链配置在系统的三个关键环节被使用：

1. **Round 计算** —— `lottery.js` 的 `computeRound` 根据 `genesisTime` 和 `period` 将用户截止时间映射为 drand round 编号。
2. **API 调用** —— `api.js` 的 `fetchBeacon` 用 `chainHash` 构建 `/chainHash/public/round` 路径，并通过 `chain.relays` 实现多 relay 故障切换。
3. **验证与展示** —— `DrawStatus.js` 根据 `params.chain` 从 `CHAINS` 中取出链信息，用于展示和校验。

[来源](../src/lottery.js#L3-L6) | [来源](../src/api.js#L14-L20) | [来源](../src/components/DrawStatus.js#L25-L28)

## 扩展新链

若需添加新的 drand 链，只需在 `CHAINS` 中新增一项，并在 `CHAIN_SHORT` / `SHORT_CHAIN` 中添加对应的单字母映射即可。`CHAIN_LIST` 数组列出了所有可用链 ID，供 UI 中的下拉选择器动态渲染。

```js
export const CHAIN_LIST = ['quicknet', 'default', 'evmnet']
```

[来源](../src/chains.js#L48)
